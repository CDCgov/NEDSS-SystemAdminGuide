# Deployment phases

NBS 7 deployment involves five phases. Bar width reflects relative effort —
actual timelines vary significantly depending on your jurisdiction's size,
existing infrastructure, staffing capacity, and security approval processes.

![NBS 7 deployment phases: Planning, Install, Test, Go-live, and Steady
state shown as horizontal bars of proportional width with key activities
listed under each phase.](../1_introduction/images/nbs7_deployment_phases.svg)

The Planning phase is often longer than expected. Security approval for cloud
hosting and required technologies — including Kubernetes, Terraform, and Docker
— is frequently the longest-lead item and the most common source of delay.
Starting that process early tends to reduce overall deployment time.

For detailed checklists and deliverables for each phase, see the
[NBS 7 Migration Info Sheet](https://nbscentral.cdc.gov/documents/731)
on NBS Central.
